Power Overview

Power Characteristic

Description

Implements characteristics as a power.  Currently, the Stat and number of points are configurable, as well as whether the power is persistent or constant, and whether the changes affect figured characteristics.

Functional Overview

When triggered, the power generates a standard effect, with one subeffect of type aid to increase the specified stat the configured amount.  The affects figured subeffect parameter is set when the effect is generated.
The configurePAD() code configures the PTYPE of the ability according to user input.

Known Caveats

-stat modification with affects figured are broken and incorrect results sometimes occur, especially when a affects figured and non-affects figured characteristic modification is used together.
-adding advantages to characteristics needs to be evaluated.
-characteristics in a mulitpower CANNOT add to figured but since MPs aren't implemented yet this option isn't implemented.
-figured characteristics can be bought back when gained from the primary characteristic. Also, this can be done if the power has a limitation but then the value needs to be proportionate to the limitation value. One of these per character only.
-need to create a limitation of -1/2 that will turn off affect of figured characteristics. When this is implemented the figured characteristic option for powerCharacteristic should be removed.
-constant option should be removed and be available as constant if "uses END" limitation is added.
