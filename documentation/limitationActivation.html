Limitation Overview

Limitation Activation Roll

Description

Implements Activatation Roll.  Currently, an activation roll is made whenever the power is activated and at the beginning of each phase for constant/persistent powers.  The roll necessary to activate is configurable, GUI checkboxes are available to tag the activation as burnout or jammable, as well as optional, Autoroll parameters for streamlining the GUI.

Functional Overview

Uses the preactivation method to present the GUI and determine if the activation succeeded.  Activation.AAUTOROLL indicates whether the initial activation rolls should be automatic.  Activation.MAUTOROLL indicates whether the reoccuring activation rolls should be automatic.  A custom action Unjam ability and Repair ability are added to the actions tab to re-enable these powers for use.  These remove the Jammed and Burned out effects which the isEnabled method in limitationActivation looks for to disable the powers.

Known Caveats

None.

Parsed Parameters

ActivationRoll

String

Activation.LEVEL
        Stores the roll necessary to activate the power, in String form.  The format is "<roll> or less", such as "8 or less".

AAutoroll

boolean

Activation.AAUTOROLL
        Indicates the initial activation roll should be made automatically.

MAutoroll

boolean

Activation.MAUTOROLL
        Indicates the reoccuring activation roll should be made automatically.

 

CanJam

Boolean

Activation.CANJAM
        Indicates that if the activation roll fails the power will be turned off until the power is unjammed.

CanBurnout

Boolean

Activation.CANBURNOUT

        Indicates that if the activation roll fails the power will be turned off until the power is repaired BUT it will activate for this one instance before turning off.
