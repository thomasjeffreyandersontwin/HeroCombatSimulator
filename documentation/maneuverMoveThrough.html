Power Overview

Maneuver Move Through

Description

This maneuver implements the standard move through maneuver.  The maneuver can be configured with additional DCs to simulate martial move throughs which cause additional damage.

Functional Overview

Move Through uses the standard BattleEngine damage generation, however, special code in the battleEngine determines if the target was knockback and applies damage to the source of the attack automatically according to the move through rules.  Preactivation() method is used to present a GUI to the user to determine the velocity of the source.  AdjustDice() and addOCVAttackModifiers() then adjust the DC and OCV of source according to that velocity.

Known Caveats

There are no known problems with move through.  However, it does have a fair bit of custom code located in the battleEngine and may be incompatible when combined with certain advantages/limitations/effects.  Currently, this is only compatible with non-AE attacks.
