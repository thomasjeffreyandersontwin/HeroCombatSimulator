Advantage Overview

Advantage Penetrating

Description

Implements the advantage penetrating.  The number of levels of penetrating are configured.

Functional Overview

Uses postpower method to guarantee that a minimum amount of damage penetrates the targets defenses.  Specifically, it looks up the roll of the first die listing in the DIE[] index, then makes sure the value of the first subeffect is at minimum equal to the body value of the first dice.  Before adjusting the subeffects, it check if there is an index Defense[].NAME equal to "Hardened", and if so, it check on Defense[].LEVELS for that index to make sure there are more Penetrating levels then hardened levels.

Known Caveats

Hardened Defenses are not properly used.  Currently having 1 point of hardened defenses allows all defense to be hardened to that level.  If multiple defense exist, only the first one will be considered, and that may not be the best one.
Since penetrating can't tell the difference between power generated effects and effect generated elsewhere (such as END), it will trigger penetrating in places it shouldn't (such as END).
